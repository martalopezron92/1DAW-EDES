## Observaciones para la Evaluación

### Aspectos clave a verificar:

**Selección de lenguajes:**
- **Grupo 1:** Python, C, C++, JavaScript, PHP, Ruby, etc. (EXCEPTO Java, Kotlin, .NET)
- **Grupo 2:** Java, Kotlin o C# (.NET)

**Programa "Hola usuario":**
```
Entrada: nombre del usuario
Salida: "Hola [usuario]. Este programa está hecho en el lenguaje de programación: [lenguaje]"
```

**Herramientas esperadas:**

*Grupo 1 - Ejemplo Python:*
- SO: Windows/Linux/macOS
- Editor: VS Code, PyCharm, Notepad++, Sublime
- Intérprete: CPython (python.exe)
- Ejecución: `python programa.py`

*Grupo 1 - Ejemplo C:*
- SO: Windows/Linux/macOS
- Editor: VS Code, Code::Blocks, Visual Studio
- Compilador: GCC, Clang, MSVC
- Enlazador: ld (integrado en compilador)
- Ejecución: `./programa` o `programa.exe`

*Grupo 2 - Ejemplo Java:*
- SO: Windows/Linux/macOS
- Editor: IntelliJ IDEA, Eclipse, VS Code
- Compilador: javac (JDK)
- Máquina Virtual: JVM (java)
- Ejecución: `java Programa`

*Grupo 2 - Ejemplo C#:*
- SO: Windows/Linux/macOS
- Editor: Visual Studio, Rider, VS Code
- Compilador: Roslyn (csc)
- Máquina Virtual: CLR (.NET Runtime)
- Ejecución: `dotnet run` o ejecutable

**Diferenciación de códigos:**

*Python (interpretado):*
- Código fuente: `programa.py` (texto legible)
- Código intermedio: `__pycache__/programa.cpython-XX.pyc` (bytecode)
- No genera ejecutable nativo tradicional

*C (compilado nativo):*
- Código fuente: `programa.c`
- Código objeto: `programa.o` (binario intermedio)
- Ejecutable: `programa.exe` o `programa` (código máquina nativo)

*Java (bytecode):*
- Código fuente: `Programa.java`
- Bytecode: `Programa.class` (código intermedio)
- No ejecutable nativo, requiere JVM

*C# (.NET):*
- Código fuente: `Programa.cs`
- Código intermedio: CIL/MSIL en `.dll` o `.exe`
- Ejecutable: Requiere CLR para ejecución

**Componentes hardware involucrados:**

1. **CPU**: Ejecuta instrucciones máquina (nativas en Grupo 1 C/C++, interpretadas/JIT en otros)
2. **Memoria RAM**: 
   - Carga del programa ejecutable
   - Stack (variables locales, llamadas a función)
   - Heap (memoria dinámica)
   - Segmento de código, datos
3. **Disco duro**: Almacenamiento permanente del ejecutable y archivos
4. **Bus de datos**: Transferencia entre CPU, RAM y periféricos
5. **Caché**: Acelera acceso a instrucciones y datos frecuentes
6. **Periféricos E/S**: Teclado (entrada nombre), pantalla (salida mensaje)

**Diferencias Grupo 1 vs Grupo 2 (Virtual Machine):**

| Aspecto | Grupo 1 (Nativo/Interpretado) | Grupo 2 (VM: JVM/CLR) |
|---------|-------------------------------|----------------------|
| **Compilación** | A código máquina directo (C) o interpretado (Python) | A bytecode intermedio |
| **Ejecución** | Directa en CPU (C) o intérprete línea a línea (Python) | Sobre máquina virtual (JVM/CLR) |
| **Portabilidad** | Dependiente de arquitectura (C) o alta (Python con intérprete) | "Write Once, Run Anywhere" |
| **Rendimiento** | Máximo (C nativo) o más lento (Python interpretado) | Intermedio, JIT optimiza |
| **Gestión memoria** | Manual (C) o automática (Python GC) | Garbage Collection automático |
| **Dependencias** | Ejecutable (C) o intérprete instalado (Python) | JVM/CLR Runtime necesario |

**Conclusiones deben incluir:**
- Ventajas de compilación nativa (rendimiento)
- Ventajas de VM (portabilidad, seguridad, gestión memoria)
- Desventajas de cada enfoque
- Casos de uso óptimos para cada tipo
- Reflexión sobre la pista "Virtual Machine"

## Guía de Puntuación Rápida

**Para obtener un 5 (Suficiente):**
- Programas funcionando en ambos grupos
- Capturas de ejecución incluidas
- Lista básica de herramientas para ambos
- Diferenciación básica entre código fuente, objeto y ejecutable
- Mención de CPU y RAM en ejecución
- Conclusión básica mencionando diferencias
- Identificación completa del trabajo

**Para obtener un 7-8 (Notable):**
- Todo lo anterior +
- Descripción completa del proceso de creación y ejecución
- Explicación clara de todos los tipos de código con nombres de archivos
- Análisis completo de componentes hardware involucrados
- Explicación de máquinas virtuales (JVM/CLR)
- Conclusiones con análisis de ventajas/desventajas
- Código limpio y bien comentado

**Para obtener un 9-10 (Sobresaliente):**
- Todo lo anterior +
- Proceso detallado con comandos específicos y flags
- Análisis profundo de formatos de archivo y contenido
- Explicación técnica de CPU (registros, ALU), memoria (stack, heap)
- Comparativa exhaustiva: portabilidad, rendimiento, JIT, GC
- Conclusiones con análisis crítico y casos de uso
- Presentación profesional con diagramas o tablas comparativas
- Código ejemplar con buenas prácticas
